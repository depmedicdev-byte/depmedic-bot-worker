# depmedic-bot

A GitHub App that audits every PR touching `.github/workflows/*.yml`.

When a PR opens (or a push updates one), the bot:

1. Lists changed files via the Pulls API.
2. Skips the PR if no workflow files changed.
3. For each changed workflow, fetches it at the PR's head SHA.
4. Runs the bundled `ci-doctor` (14 rules) against the YAML.
5. Posts (or updates) a single PR comment with a markdown table of findings.

Runs on Cloudflare Workers. ~3 GitHub API calls per PR delivery. Free
tier covers ~100k PRs/mo.

## Files

- `wrangler.toml` - Worker config + KV binding for installation IDs.
- `src/worker.js` - the webhook handler + JWT minter + comment renderer.
- `src/ci-doctor-bundled.js` - generated by `npm run build:ci-doctor`,
  exports `auditWorkflow` from the local `projects/ci-doctor` package.
- `app-manifest.json` - the GitHub App manifest the user pastes when
  creating the App via the UI flow (one click).
- `tools/build-ci-doctor-esm.mjs` - regenerates the bundled audit
  engine when ci-doctor changes upstream.

## One-time setup (user-gated, ~10 min)

1. **Create the Worker KV namespace.**
   ```powershell
   wrangler kv namespace create depmedic-bot-installs
   ```
   Paste the returned id into `wrangler.toml` (`INSTALLS.id`).

2. **Create the GitHub App.**
   - Visit <https://github.com/settings/apps/new>.
   - Use `app-manifest.json` to populate fields (paste, do not retype).
   - Webhook URL: `https://depmedic-bot.depmedicdev.workers.dev/webhook`
     (the worker URL after deploy).
   - Generate + download a private key (PKCS#8 PEM).
   - Generate a webhook secret (`openssl rand -hex 32`).
   - Note the App ID.

3. **Set secrets and vars.**
   ```powershell
   wrangler secret put GH_PRIVATE_KEY     # paste the private key (escape newlines as \n)
   wrangler secret put GH_WEBHOOK_SECRET  # the secret you generated
   # Then set APP_ID inline in wrangler.toml [vars]
   ```

4. **Deploy.**
   ```powershell
   npm run build:ci-doctor    # writes src/ci-doctor-bundled.js
   wrangler deploy
   ```

5. **Install on a test repo.**
   - Visit `https://github.com/apps/depmedic-bot`
   - Install on `depmedicdev-byte/ci-doctor` (or any repo you own).
   - Open a test PR that edits `.github/workflows/test.yml`.
   - The bot should comment within ~10 seconds.

## Marketplace listing (D.2, separate setup)

After verifying the App works on a few repos:

1. <https://github.com/marketplace/manage> -> "List a new app".
2. Pick `depmedic-bot`.
3. Two pricing plans: `Public repos free`, `Private repos $7/mo per repo`.
4. Submit for review (~5-7 business days, GitHub does not charge a
   listing fee but requires a verified publisher - the user does
   verification once, in the GitHub UI).

Once approved, set `ALLOWED_PRIVATE_REPOS = "true"` in `wrangler.toml`
and redeploy. The Worker code is already gated.

## Cost

Cloudflare Workers free tier: 100k requests/day. GitHub API calls are
free. Total monthly cost at any reasonable PR volume: $0.

## Security model

- Webhook signature is verified with HMAC-SHA256 against
  `GH_WEBHOOK_SECRET` before any work is done.
- Private key is stored as a Worker secret (not a var); never logged.
- Installation tokens are minted per request and never persisted.
- No PR diff or file content leaves the Worker; only the audit findings
  (rule id + line + message, all extracted from your own YAML) end up
  in the PR comment.
